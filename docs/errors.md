# LinkPulse API Errors and RFC7807 Specification

All API errors generated by LinkPulse utilize the standardized **RFC7807 Problem Details** (`application/problem+json`) format. This structure is designed to be fully backward compatible with older clients parsing standard envelopes while exposing modern developer details.

---

## 1. RFC7807 Response Schema

Every API error response returns the following JSON object structure:

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `success` | `boolean` | Flag value always set to `false`. |
| `error` | `object` | Backward-compatible wrapper containing `code` and `message`. |
| `type` | `string` | Absolute URI identifying the category of error. |
| `title` | `string` | Short human-readable summary of the error type. |
| `status` | `integer` | The HTTP status code generated by the origin server. |
| `detail` | `string` | Detailed, contextual explanation of this specific error occurrence. |
| `instance` | `string` | Relative URI reference of the specific request path that failed. |
| `request_id` | `string` | Traceability UUID matching correlation logs (X-Request-ID). |

---

## 2. Standard Error Catalog

Here are the registered API error codes and their mapped RFC7807 schemas:

### Validation Error (`422 Unprocessable Entity`)
Returned when request parameters, query values, or body parameters fail format constraint specifications.

**Type**: `https://linkpulse.com/errors/validation-error`  
**Title**: `Validation Error`  
**Response JSON**:
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validation failed"
  },
  "type": "https://linkpulse.com/errors/validation-error",
  "title": "Validation Error",
  "status": 422,
  "detail": "One or more fields failed validation constraints.",
  "instance": "/api/v1/links",
  "details": [
    {
      "field": "original_url",
      "rule": "url",
      "message": "The original_url field must be a valid absolute URL (http or https)"
    }
  ],
  "request_id": "8dfa0f62-56a7-a8d3-b56a-688b9ffbdac0"
}
```

### Invalid Credentials (`401 Unauthorized`)
Returned on failed login requests when the username or password does not match.

**Type**: `https://linkpulse.com/errors/invalid-credentials`  
**Title**: `Invalid Credentials`  
**Response JSON**:
```json
{
  "success": false,
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "Invalid email or password"
  },
  "type": "https://linkpulse.com/errors/invalid-credentials",
  "title": "Invalid Credentials",
  "status": 401,
  "detail": "Invalid email or password",
  "instance": "/api/v1/auth/login",
  "request_id": "8dfa0f62-56a7-a8d3-b56a-688b9ffbdac0"
}
```

### Resource Gone / Expired (`410 Gone`)
Returned when resolving a shortened code that has reached its configured expiration date.

**Type**: `https://linkpulse.com/errors/resolve-failed`  
**Title**: `Resolve Failed`  
**Status**: `410`  
**Detail**: `Short code has expired`
